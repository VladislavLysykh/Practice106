# Лабораторная работа №106
## Continuous Integration (CI)

### Цель работы
Настроить процесс автоматического тестирования проекта при создании Pull Request.

### Используемый проект
Проект из лабораторной работы №105:
сервис анализа текста на основе микросервисной архитектуры Docker Compose.

Состав проекта:

- Frontend
- Backend (Flask API)
- Worker
- Nginx

### Используемая CI/CD система

GitHub Actions

Файл workflow:

.github/workflows/ci.yml

### Pipeline выполняет

1. Запуск при Pull Request в ветку main
2. Установка Python
3. Установка зависимостей
4. Запуск тестов pytest

### Проверка успешного выполнения

Создана ветка:

success-ci

Выполнен Pull Request.

Тесты проходят успешно.

### Проверка ошибочного выполнения

Создана отдельная ветка с намеренно сломанным тестом.

Pipeline завершается ошибкой.

### Результат

Настроен CI-процесс на GitHub Actions.

При каждом Pull Request автоматически выполняются тесты проекта.

Успешные изменения проходят проверку, ошибки выявляются автоматически.
